# Welight 官网一页式重构 Prompt

下面这段 prompt 可以直接交给 Codex，用于重构 Welight 官网。

```md
你是资深前端设计工程师。请基于当前官网代码，对 Welight 官网进行一次完整的一页式重构。参考站点：https://mole.fit/ ，但不要复制它的文案、星球意象、动效素材或具体视觉资产。目标是学习它的页面组织方式：极简、产品感强、首屏信息密度高、分段清晰、价格透明、FAQ 直接回答问题。

## 任务目标

将 Welight 官网改造成一个清晰、克制、可信的一页式产品官网，重点服务三类用户：

1. 第一次听说 Welight 的公众号创作者，能在首屏理解“它解决什么问题”。
2. 正在比较排版工具、AI 写作工具、公众号编辑器辅助工具的人，能快速看到差异点。
3. 已准备下载或购买的人，能在页面内直接完成下载、了解价格、查看常见问题。

重构完成后，首页应是产品本身，而不是营销空话。不要做大段品牌宣言，不要堆装饰卡片，不要做泛 AI 落地页。

## 当前产品信息

请先阅读当前仓库中的 README、系统文档、更新日志和官网相关代码，确认产品信息后再动手。已知产品事实如下：

- 产品名：Welight。
- 定位：面向微信公众号创作者的桌面写作、AI 创作与一键排版工具。
- 官网域名：`https://waer.ltd`。
- 核心价值：把 Markdown / 普通文本实时渲染为适合微信公众号的图文排版，并支持一键复制到公众号后台。
- 技术形态：Tauri 桌面应用，前端 Vue 3 + TypeScript，支持 macOS、Windows、Linux。
- 基础编辑能力：Markdown、数学公式、Mermaid 图表、PlantUML、Ruby 注音、代码高亮、主题样式、自定义 CSS。
- 排版能力：实时预览、多主题切换、智能排版、一键复制、导入导出、公众号发布前处理。
- 图片能力：图床配置、图片上传、公众号素材、封面图/配图相关能力。
- AI 能力：AI 创作页面、会话侧边栏、流式输出、工具调用、任务步骤、推理过程、上下文、检查点、联网搜索、记忆、工作空间、模型配置、AI 生图。
- 模型/服务支持：DeepSeek、OpenAI、通义千问、腾讯元宝、豆包、讯飞星火、302.AI、SiliconFlow、OpenRouter、可灵等。以代码和配置中的真实支持为准。
- 定价信息需要谨慎处理：本地文档中出现过“终身许可证 14.99 元”和积分套餐，但更新日志显示 v5.0.0 已移除积分/Credits 系统，官网搜索结果也提示积分功能将于 2026 年 6 月底关停。请不要把积分套餐作为主要售卖项。页面应以“买断许可证 / 试用 / 自配 AI 服务或官方当前可用服务”为主，并在实现前核对当前官网或配置中真实价格。

## 参考 mole.fit 的方式

请参考 Mole 的信息架构和节奏，而不是照搬视觉符号：

- 顶部导航极简：Logo、产品名、少量锚点、下载/购买按钮。
- 首屏直接讲清楚产品：一句强标题、一句副标题、价格/平台/试用信息、主次 CTA、产品截图。
- 页面按编号分段：`00`、`01`、`02`、`03`、`04`，让浏览有秩序感。
- 每个能力模块都绑定真实使用场景，而不是抽象功能词。
- 有一段“产品行为原则”：本地优先、可控、先预览再操作、不强迫订阅、不假装全自动。
- 价格区直接透明。
- FAQ 用诚实问答，不写销售腔。
- 页脚保留下载、文档、更新日志、反馈、隐私、条款等入口。

## 建议页面结构

### 顶部导航

保留一行固定或半固定导航：

- 左侧：Welight Logo + Welight。
- 中间：`功能`、`工作流`、`AI 创作`、`定价`、`FAQ`。
- 右侧：`下载试用`、`购买许可证`。

移动端导航要简洁，不要占用首屏高度。

### 00 · 首屏

首屏应在第一屏内完成理解和转化。

建议文案方向：

- H1：`公众号写作、AI 创作和排版，放在一个干净的桌面应用里`
- 副标题：`Welight 把 Markdown、实时预览、主题排版、AI 创作和一键复制整合在一起，让公众号文章从选题到发布前排版少切几次工具。`
- 信息短条：`macOS / Windows / Linux`、`Tauri 桌面应用`、`一键复制到公众号`、`支持自配 AI 模型`
- CTA：
  - 主按钮：`下载试用`
  - 次按钮：`查看价格`

首屏视觉必须展示真实产品界面或高质量产品截图，不要使用抽象插画。可以用一张主截图表现“左侧写作 + 右侧公众号预览 + AI 创作入口”的核心体验。

### 01 · 四个核心工作区

用 4 个横向或网格模块展示 Welight 不是单点工具，而是一套公众号创作工作流。

1. `写作`
   - Markdown / 普通文本输入。
   - 代码、公式、图表、注音等内容结构支持。
2. `排版`
   - 实时公众号预览。
   - 多主题、一键智能排版、自定义 CSS。
3. `AI 创作`
   - 选题、扩写、润色、摘要、解释、表格、配图、封面图等能力。
   - 多会话、流式输出、工具调用、联网搜索、记忆和工作空间。
4. `发布前处理`
   - 图床上传。
   - 一键复制到公众号后台。
   - 导入导出、素材整理、发布前检查。

每个模块都要有短标题、两三句说明和一个小型界面细节图/截图/示意。不要做大而空的图标卡片。

### 02 · 一条真实工作流

做一个从左到右或纵向的流程区：

`写下初稿` -> `交给 AI 补全结构` -> `选择主题排版` -> `上传图片/生成封面` -> `一键复制到公众号`

每一步都写具体结果，不写“提升效率”“赋能创作”这类空话。

### 03 · 产品行为原则

参考 Mole 的 “Conservative by default”，为 Welight 写成更适合创作工具的原则区。

建议四条：

1. `本地写作优先`
   - 桌面应用提供独立、干净的写作环境，避免在公众号后台反复调样式。
2. `先预览，再发布`
   - 所见即所得预览，复制前能看到公众号图文效果。
3. `AI 是协作者，不是黑盒`
   - 展示流式过程、工具调用、任务步骤和可保存产物，避免用户不知道发生了什么。
4. `能自配，就不锁死`
   - 支持多家模型和图床配置，尽量把选择权交给用户。

### 04 · 定价

做一个简洁透明的定价区。

要求：

- 如果代码或当前官网能确认价格，展示真实价格。
- 如果不能确认，不要硬编码旧价格；使用“以当前购买页为准”的文案，并把 CTA 指向购买页。
- 不再把积分套餐作为主要定价内容。
- 可以强调：一次购买、桌面应用、试用、后续更新策略、支持平台。

建议卡片：

- `试用`
  - 下载即可体验核心功能。
  - 适合先确认是否满足公众号排版工作流。
- `个人许可证`
  - 买断使用，价格以当前购买页为准。
  - 适合长期写公众号文章的个人创作者。

### 05 · FAQ

FAQ 要直接回答用户真实疑问：

- `Welight 和公众号后台编辑器有什么区别？`
- `我不会 Markdown 能用吗？`
- `AI 功能必须购买官方服务吗？`
- `能不能一键复制到公众号？`
- `支持哪些系统？`
- `图片和文章内容会上传到哪里？`
- `和 doocs/md 是什么关系？`
- `试用和许可证怎么激活？`

答案要短，诚实，避免绝对化承诺。

### 页脚

页脚包含：

- Welight Logo 和一句产品描述。
- 下载、定价、更新日志、文档、反馈、隐私、条款、GitHub 或相关链接。
- 版权信息。

## 视觉方向

整体风格：克制、轻量、桌面软件感、内容优先。

建议：

- 背景：接近白色或浅暖灰，避免大面积渐变。
- 字体：使用系统 sans 字体；中文保持清晰，不要使用花哨字体。
- 色彩：黑白灰为主，搭配 Welight 品牌色或一个低饱和强调色。不要做满屏蓝紫渐变。
- 卡片：少用大卡片；如果使用，边框浅、圆角不超过 8px。
- 间距：桌面端内容最大宽度约 1120-1200px，段落之间有明确呼吸感。
- 动效：只保留轻微入场、悬停、截图切换等，不要用夸张滚动动画。
- 图片：优先使用真实产品截图。若当前仓库没有官网素材，请从应用截图、已有文档图片或静态资源中选取；不要用无关 stock 图。

## 实现要求

1. 先定位当前官网入口、路由、组件和样式组织方式。
2. 尽量沿用现有技术栈和组件风格，不引入大型新依赖。
3. 如果项目使用 Vue，优先拆成语义清晰的小组件：`HeroSection`、`FeatureSection`、`WorkflowSection`、`PrinciplesSection`、`PricingSection`、`FAQSection`、`SiteFooter`。
4. 文案集中在同一个数据结构或常量里，方便后续调整。
5. 所有 CTA 链接使用当前项目已有下载、购买、文档、反馈地址；如果找不到，保留 TODO 注释并在最终说明里列出。
6. 保证移动端首屏不拥挤，按钮不换行溢出，截图不压扁。
7. 保证可访问性：按钮/链接语义正确、图片有 alt、颜色对比足够。
8. 不要删除无关页面和业务逻辑。

## 验收标准

- 代码格式检查或项目已有 lint。
- 类型检查或构建命令。
- 启动本地开发服务器。
- 用浏览器分别检查桌面和移动端视口：
  - 首屏能看到产品名、主标题、CTA 和产品截图。
  - 导航锚点可跳转。
  - 价格区不展示过期积分套餐。
  - FAQ 可读。
  - 无明显横向滚动、遮挡、文字溢出。

最终回复请包含：

- 修改了哪些文件。
- 采用了哪些产品文案和页面结构。
- 哪些价格/下载/购买链接需要我确认。
- 执行过的验证命令和结果。
```
